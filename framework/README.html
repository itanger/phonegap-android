Creating a new BONDI-Android-Application

 

The simplest way to create a new BONDI-Android-application is to build up the application based on the example-Application.

 

An application consists of the manifest file (AndroidManifest.xml), the initial html page (e.g. assets/www/index.html) and the resources (icons, labels).

 

For creating a new application based on the example application the following steps are recommended:

(1) Create a new html-page that loads phonegap.js. The html page is the start page of the application

(2) Adapt the manifest file: Define a adequate label android:label="@string/app_name_bonditest"and an icon android:icon="@drawable/icon". As initial activitiy the class activity android:name=".DroidGap" can be used. Make sure the permissions are set correctly.

(3) The initial html page, loaded by the class "DroidGap" is defined in the file "res/values/strings.xml". To change the initially loaded page, the entry <string name="url">file:///android_asset/www/indexBondi.html</string> must be changed.

 

NOTE: The Class DroidGap defines a JavaScript-Java-Binding for all objects used by the BONDI-js-functions. The JavaScript code required by BONDI is completely provided by the file phonegap.js. The have access to the BONDI-functionality it is only necessary to include the javascript file phonegap.js (<script type="text/javascript" charset="utf-8" src="phonegap.js"></script>).
